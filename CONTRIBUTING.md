# Contributing to SwitchHosts

感谢你愿意参与 SwitchHosts 的开源建设。

## 开始之前

- 阅读 [README.md](README.md) 了解项目定位与运行方式。
- 提交代码前请确保本地可编译、可运行。
- 变更应聚焦单一目标，避免夹带无关重构。

## 提交流程

1. Fork 本仓库并创建功能分支：`feat/xxx`、`fix/xxx`。
2. 完成代码修改并进行自测。
3. 提交 Pull Request，说明：
   - 变更背景
   - 主要改动
   - 验证方式
4. 等待 Code Review，并根据意见迭代。

## 代码风格

- 遵循现有 Swift / SwiftUI 代码风格。
- 命名清晰、职责单一，优先保持可读性。
- 不在一次 PR 中处理无关问题。

## Issue 规范

- Bug 反馈请尽量提供：复现步骤、预期行为、实际行为、运行环境（macOS / Xcode 版本）。
- 功能建议请说明使用场景与收益。

## 行为准则

参与本项目即表示同意遵守 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)。
